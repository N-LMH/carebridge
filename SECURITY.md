# Security Policy

## Scope

CareBridge is a hackathon MVP and demo repository. It is intended for evaluation, learning, and local demonstration use.

It is not a production medical system and must not be used to store, process, or transmit real sensitive medical data without additional security, privacy, and compliance work.

## Supported Versions

Only the latest code on the `main` branch is considered supported for security fixes.

## Reporting a Vulnerability

If you discover a security issue, please avoid posting sensitive exploit details in a public issue.

Instead, contact the repository owner directly through GitHub first and include:

- A short description of the issue
- Steps to reproduce it
- The affected file, route, or feature
- Any suggested mitigation, if known

Please do not include real patient data, credentials, tokens, or other secrets in any report.

## Security Notes for This Project

- This project uses local JSON storage for MVP persistence.
- Demo and test data only should be used in the repository.
- Outputs are for pre-visit guidance only and are not a substitute for clinical systems or formal diagnosis.
